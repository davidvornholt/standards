# Phase 2

Goal: codify the proven vibe into shared design intent and reusable theme primitives.

- Create or update the root `DESIGN.md`.
- `DESIGN.md` should describe how the design should feel, not list raw token values, exact component inventories, or CSS implementation details.
- Create `packages/ui` if it does not already exist.
- Create or update `packages/ui/src/theme.css`.
- Turn the successful prototype into a reusable theme system instead of copying page-specific styling verbatim.
- Keep `DESIGN.md` and `theme.css` aligned.

**In scope:** design intent, token ramps, semantic aliases, theme import, and base-layer/shell styling.

**Out of scope:** migrating feature pages, shared components, or prototype-specific classes across the app. That belongs in Phase 3.

This phase should preserve the spirit of the prototype while making the system reusable.
