# Phase 1

Goal: create the strongest possible prototype.

- Use the existing design direction as orientation, not as a cage. If `DESIGN.md` exists at the repo root, skim it for voice and palette intent.
- Use the `frontend-design` skill to make it look exceptional and best-in-class.
- If the user asks for a specific style direction, check `references/styles/` for a matching style guide and use it as creative direction for the prototype.
- Ignore the current shared components and the current `packages/ui/src/theme.css` as design constraints while exploring.
- Prioritize originality, taste, cohesion, and visual force.
- Do not prematurely optimize for systemization or reuse in this phase.

Finish with the best version of the prototype the request calls for.
