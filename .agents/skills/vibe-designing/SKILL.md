# Vibe designing

Use this skill to route design work through the right phase without turning the workflow into a rigid checklist.

Read only the phase reference you need:

- Broad design prompts, redesigns, or requests for a beautiful best-in-class concept usually start in [references/phase-1.md](references/phase-1.md).
- Requests about `DESIGN.md`, design feel, theme codification, or shared theme work go to [references/phase-2.md](references/phase-2.md).
- Requests to clean up, componentize, reuse, or productionize an existing prototype go to [references/phase-3.md](references/phase-3.md).
- If the user asks for a specific style direction, read the matching file in `references/styles/` when one exists.

If the user clearly asks for multiple phases, handle them in order.
